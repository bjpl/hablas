- Pauses between phrases
- Repetition of key terms
- ~120-140 words per minute

- Standard pauses
- ~150-180 words per minute

**Avanzado (Advanced):** 1.0-1.1x speed (normal to slightly faster)
- Professional business pace
- Natural flow

## Technical Implementation

### Batch Generation Script Structure

```bash
#!/bin/bash
# scripts/generate-audio-batch.sh

# Phase 1: High Priority (12 files)
PHASE1_IDS=(22 23 27 29 48 50 1 4 6 11 14 17)

# Phase 2: Medium Priority (9 files)
PHASE2_IDS=(5 31 16 33 25 26 45 46 51)

# Phase 3: Low Priority (5 files)
PHASE3_IDS=(35 36 39 53 57)

# Voice rotation pattern
declare -A VOICE_MAP
VOICE_MAP[22]="mx_spanish"
VOICE_MAP[23]="co_spanish"
VOICE_MAP[27]="us_english"
# ... (full mapping in implementation)

# Generate with rate adjustment
for id in "${PHASE1_IDS[@]}"; do
  voice="${VOICE_MAP[$id]}"
  rate=$(get_rate_for_level "$id")  # Returns 0.9 for básico, 1.0 for intermedio, 1.1 for avanzado

  node scripts/generate-audio.js \
    --resource-id "$id" \
    --voice "$voice" \
    --rate "$rate" \
    --output "public/audio/resource-${id}.mp3"

  # Update metadata.json
  node scripts/update-audio-metadata.js \
    --resource-id "$id" \
    --voice "$voice" \
    --file "/audio/resource-${id}.mp3"
done
```

### Quality Checks

**Pre-Generation:**
1. Validate source text exists (contentPath field)
2. Verify voice availability in gTTS
3. Check audio directory permissions
4. Confirm resource ID not already in metadata

**Post-Generation:**
1. File size validation (>100KB, <10MB)
2. Duration check (1.5-6 minutes expected)
3. Audio format verification (MP3, 44.1kHz)
4. Bitrate check (64-128 kbps)
5. Silence detection (no files with >50% silence)

**Manual Spot Checks:**
- Listen to first and last 10 seconds
- Check pronunciation clarity
- Verify pacing matches level
- Ensure no audio artifacts

## Risk Mitigation

### Technical Risks
**Risk:** gTTS API rate limiting
**Mitigation:** Add delays between requests (2-3 seconds), implement retry logic

**Risk:** Audio quality variations
**Mitigation:** Standardize generation parameters, post-process normalization

**Risk:** Storage limitations
**Mitigation:** Compress files appropriately (64kbps sufficient for speech)

### Content Risks
**Risk:** Incorrect pronunciation in source text
**Mitigation:** Review by native speakers before generation

**Risk:** Inappropriate content selection
**Mitigation:** Follow priority framework, validate with stakeholders

### Operational Risks
**Risk:** Timeline delays
**Mitigation:** Prioritized phases allow partial delivery

**Risk:** Quality issues discovered post-deployment
**Mitigation:** Staged rollout, easy rollback capability
